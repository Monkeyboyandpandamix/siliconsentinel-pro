AUDIT METHOD REQUIREMENTS
- Base conclusions on code evidence, not README optimism.
- Do not assume a feature is real just because a file exists.

- If evidence is missing, say “unproven.”
- Call out contradictions clearly.
- Prefer precise criticism over vague advice.
- Do not rewrite the project. Audit it.
